# 秩序尖塔 · ORDER TOWER — 设计文档

> 一款取材《Side Order》的**色彩肉鸽 · 卡牌构筑爬塔**（roguelike deckbuilder）文字游戏。
> 姊妹项目：《青史另说》《SPLAT》

## 一、一句话
你和 8 号一起攀登一座把一切个性抹成纯白的「秩序之塔」，逐层用**赤・蓝・黄・紫**四色芯片构筑能力、对抗被秩序同化的敌人；你的**调色板配比**决定你成为怎样的存在，直至塔顶决战。

## 二、世界观 / 角色
被「秩序」抹除色彩的白色世界、平行卡农（塔主/终盘 Boss）、8 号（同行者，可与之和解）、依达 / 小姬（电台指引，风味）。日文名一律中文化。

## 三、核心循环
```
开场 → 逐层向上（第1~12层）
     每层：三选一路线（战斗/精英/补给站/异常信号/整备点）
       战斗：回合制，用手牌芯片，注意同色连携
       胜利：三选一收一枚芯片；获得记忆碎片
       补给站：用碎片买芯片/治疗/删卡
       异常信号：随机事件（含「与8号和解」分支）
       整备点：回血 或 重铸(强化)芯片
     第12层 → 塔顶决战「平行卡农」
血耗尽=被同化(白色结局)；登顶后按调色板配比结算结局
```

## 四、四色调色板（= 四种战斗流派 / 评分轴）
| 色 | 流派 | 机制取向 |
|---|---|---|
| **赤** | 进攻 | 高伤害、连斩、爆发 |
| **蓝** | 防御 | 护盾、反弹、稳态回血 |
| **黄** | 辅助 | 回复、抽牌、加墨力、强化 |
| **紫** | 干扰 | 侵蚀(毒)、虚弱、汲取 |

**同色连携**：芯片的 `scale` 效果——本回合每多打出一张同色芯片，效果额外叠加（鼓励单色爆发流），与「均衡多彩」形成取舍。

## 五、战斗系统
- 玩家：HP 50、每回合墨力 3、手牌 5。抽牌堆/弃牌堆循环。
- 护盾每回合清零；侵蚀(玩家)在回合开始扣血并 -1。
- 敌人有**意图预告**（攻击/侵蚀/强化/护盾），循环行动；虚弱使其攻击 -40%。
- 敌人侵蚀(紫毒)在你回合末结算掉血。
- 敌人随层数成长（HP、攻击力递增）；精英更强、Boss 140 血多段意图。

## 六、芯片库（20 枚 + 强化版）
赤5 / 蓝5 / 黄5 / 紫5，含基础、缩放(scale)、爆发、汲取、净滤等。起始牌组 8 张（3赤3蓝1黄1紫）。奖励与商店从池中三选一。整备点可 `重铸` 让某芯片数值 +50%。

## 七、结局（7 种，可图鉴收集）
| 结局 | 触发 |
|---|---|
| **还世界以色彩** | 登顶 + 调色板均衡（无单色≥45%）——正典好结局 |
| **铁血/壁垒/辉光/诡影 塔主** | 登顶 + 某色主导(≥45%)——赤/蓝/黄/紫 四变体 |
| **与 8 号和解** | 触发「8号的低语」事件选择伸手 + 登顶 |
| **褪为白色** | 血耗尽被同化 |

结局用 localStorage 收集，结果卡显示「结局图鉴 N/7」。可保存结局卡长图。

## 八、文件
`index.html` / `style.css` / `data.js`(芯片·敌人·事件·结局·配置) / `game.js`(引擎) / `qrcode.min.js` `html2canvas.min.js`。纯静态、无构建、可微信内直玩。

## 九、平衡
以启发式机器人跑多局校准：目标胜率约 30–60%（肉鸽应有难度但可通关），四色主导结局与均衡结局都可达。数值可在 `data.js` 顶部与 `CONFIG` 快速调整。
